## محدوده MVP پروژه مهر سلیمان

### موارد شامل (INCLUDED)
- صفحات اصلی دمو با محتوای استاتیک: خانه، خدمات، اخبار، درباره، تماس.
- هدر/فوتر مشترک طبق دستورالعمل طراحی (کانتینر ثابت، فونت وزیرمتن، پالت تاییدشده).
- جریان‌های نمایشی ساده برای ثبت‌نام/ورود و وضعیت کاربر با داده ماک (LocalStorage).
- داشبوردهای نمونه (کاربر/ادمین) با داده ماک برای نمایش ساختار و جریان‌ها، بدون منطق تجاری واقعی.
- گرید و فاصله‌بندی مبتنی بر موبایل‌فرست و کانتینر ۱۶/۲۴/۱۲۰۰px بر اساس مستندات طراحی.
- تایپوگرافی وزیرمتن با وزن‌های مجاز (Regular/Medium/Bold) و بولد هدفمند (تیتر، CTA، اعداد کلیدی).

### موارد خارج از محدوده (EXCLUDED)
- هرگونه اتصال به API واقعی، بک‌اند، پایگاه داده یا سرویس ابری.
- هر نوع پرداخت یا درگاه بانکی (واقعی یا سندباکس).
- اتوماسیون ایمیل/پیامک/اعلان، لاگین اجتماعی یا یکپارچه‌سازی هویت.
- منطق پیچیده کسب‌وکار، گزارش‌گیری زنده، یا سیستم نقش/مجوز چندلایه.
- انیمیشن‌های سنگین یا ری‌برندینگ خارج از پالت و تایپوگرافی مصوب.

### داده ماک و شبیه‌سازی
- تمام داده‌ها ماک هستند و درون مخزن یا LocalStorage ذخیره می‌شوند؛ هیچ درخواست شبکه واقعی ارسال نمی‌شود.
- سناریوهای ورود/خروج، منوی کاربر، فهرست خدمات/اخبار صرفاً نمایشی و بر پایه داده ثابت هستند.
- هر تغییر داده در UI موقت است و تضمین پایداری یا ذخیره‌سازی دائمی ندارد.

### منع یکپارچه‌سازی واقعی
- عدم استفاده از کلید API، Webhook، OAuth یا سرویس شخص ثالث.
- هیچ پرداخت، کارت بانکی یا تراکنش مالی—even sandbox—پیاده‌سازی نمی‌شود.
- هرگونه اشاره به اتصال بیرونی باید به‌صراحت به‌عنوان «ماک/نمایشی» برچسب بخورد.

### هدف MVP
- ارائه نسخه نمایشی برای نمایش طراحی، جریان‌های کاربری و ساختار اطلاعاتی.
- کمک به تصمیم‌گیری محصولی و هم‌راستاسازی ذینفعان پیش از توسعه بک‌اند/یکپارچه‌سازی.
- فراهم‌کردن بستری برای تست مفهومی (concept validation) بدون ریسک‌های امنیتی یا مالی.
