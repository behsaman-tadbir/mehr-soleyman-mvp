## موارد غیرقابل مذاکره
- [ ] فونت اصلی «وزیرمتن» با بولد هدفمند و محدود (تیتر و CTA)، بدون جایگزین غیرمصوب.
- [ ] طراحی Mobile-first با پاسخ‌گویی کامل در موبایل، تبلت و دسکتاپ.
- [ ] استفاده از کانتینر هم‌تراز برای تمام صفحات اصلی با فاصله‌های چپ/راست یکسان.
- [ ] این پروژه صرفاً دمو MVP است؛ هیچ API واقعی، پرداخت واقعی یا بک‌اند نباید در مستندات یا ماک‌ها القا شود.
- [ ] پایبندی به پالت آموزشی بازیگوش تاییدشده؛ از رنگ CTA فقط برای دعوت به اقدام استفاده شود و تغییری در پالت داده نشود.

## استاندارد کانتینر و گرید
- [ ] پدینگ اینلاین موبایل: ۱۶px
- [ ] پدینگ اینلاین تبلت: ۲۴px
- [ ] دسکتاپ: حداکثر عرض ۱۲۰۰px، پدینگ اینلاین ۲۴px، کانتینر وسط‌چین.
- [ ] ارتفاع هدر: ۵۶px در موبایل، ۶۴px در دسکتاپ.

## مشخصات هدر
- [ ] چینش دسکتاپ: برند در راست، ناوبری (حداکثر ۵ آیتم: خانه، خدمات، اخبار، درباره، تماس) در مرکز، اکشن‌ها در چپ.
- [ ] اکشن‌ها: CTA ورود/ثبت‌نام در حالت لاگ‌اوت؛ منوی کاربر در حالت لاگین (شبیه‌سازی با LocalStorage در دمو).
- [ ] موبایل: آیکن همبرگری + منوی کشویی؛ CTA قابل‌مشاهده داخل منو یا ناحیه اکشن.
- [ ] هدر چسبان (sticky) با حالت اسکرول (مثلاً سایه یا تغییر پس‌زمینه) و وضعیت فوکوس/هوور قابل‌دسترس.
- [ ] استفاده از تگ‌های semantic: `<header>`، `<nav>`، متن جایگزین (alt) برای لوگو.

## مشخصات فوتر
- [ ] دسکتاپ: ۳–۴ ستون (برند، لینک‌های اصلی، پشتیبانی/اطلاعات، در صورت نیاز شبکه اجتماعی/تماس).
- [ ] موبایل: چینش پشته‌ای یا آکاردئونی برای خوانایی.
- [ ] نوار پایینی: خط جداکننده + کپی‌رایت/سال، هم‌تراز با کانتینر.
- [ ] استفاده از `<footer>` با حالت فوکوس/هوور مناسب و رعایت کنتراست از پالت تاییدشده.

## نکات معنایی و سئو
- [ ] به‌کارگیری تگ‌های `<header>`، `<nav>`، `<footer>` و ساختار منطقی هدینگ‌ها.
- [ ] متن جایگزین برای لوگو و تصاویر ضروری.
- [ ] حالت فوکوس قابل‌مشاهده برای لینک‌ها و کنترل‌ها؛ رعایت ترتیب تب‌پذیری.

## استثناهای داشبورد
- [ ] داشبورد کاربر: هدر/تاپ‌بار اختصاصی؛ فوتر حداقلی یا حذف شود.
- [ ] داشبورد ادمین: سایدبار + تاپ‌بار؛ فوتر حذف شود.
- [ ] سایر صفحات اصلی حتماً از هدر/فوتر مشترک تبعیت کنند و کانتینر ثابت داشته باشند.

## چک‌لیست QA
- [ ] تراز کانتینر و فاصله‌های چپ/راست در تمام صفحات اصلی رعایت شده است.
- [ ] ناوبری موبایل (همبرگر + دراور + CTA) بدون تداخل و قابل استفاده است.
- [ ] رنگ CTA فقط در دکمه دعوت به اقدام به‌کار رفته و با پالت تاییدشده هماهنگ است.
- [ ] هیچ اشاره‌ای به شبکه/پرداخت واقعی یا اتصال بک‌اند در UI دیده نمی‌شود (ماهیت دمو حفظ شده است).
